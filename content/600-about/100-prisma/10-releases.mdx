本页解释了 Prisma 的发布进程，它是如何版本控制以及如何处理在整个发布过程中可能发生的破坏性更改。

</TopBlock>

## 发布管理

Prisma 每两周发布一次。请注意，这不是一个硬性规则-发布可能会因内部原因推迟。

[查看 GitHub 中的所有版本说明](https://github.com/prisma/prisma/releases).

## 产品成熟度级别

一次发行可以包括很多不同成熟界别的产品或者功能。成熟度级别描述了产品或功能的完整性以及用户在更改方面的期望。

> **注意**: 版本[2.13.0](https://github.com/prisma/prisma/releases/2.13.0), '实验' 不再是产品成熟度量表的一部分。

### 早期版

如果一项产品或者功能是**早期版**：

- 我们已经验证了一个问题并正在考虑解决方案，但不确定该解决方案是完整的还是完美的。
- 我们希望收集更多反馈并根据需要调整解决方案，因为我们知道用户已经为重大突破性变化做好了准备。

我们不建议在生产环境中使用早期版的产品或者功能。

### 预览版

如果一项产品或者功能是**预览版**:

- 我们已经在方向和表层上验证了产品或者功能。
- 除非版本说明和文档中另有说明，否则用户能使用功能或者产品以及关联的 API。
- 没有重大的已知问题，尽管可能存在小错误。
- 我们欢迎对这些问题的反馈，以使解决方案尽快稳定。

预览通常在功能标志后面使用或者需要某种形式的选择加入 (例如, cli 命令 `--preview-feature` 或者 [添加到 Prisma schema 的 Prisma 客户端中`generator`区块中的`previewFeatures`属性](../../concepts/components/preview-features/cli-preview-features/) )。

我们不建议在生产环境中使用预览的产品或者功能。

另请参阅: [所有当前预览功能](../../concepts/components/preview-features).

### 正式版 (GA)

如果一项产品或者功能是**正式版**:

- 该解决方案已经测试了一段时间，我们收到了足够的反馈，认为它很稳定，可以投入生产使用。
- 99%的情况下应该没有 bug(不能保证完全没有 bug 的软件).

## 版本控制

在`3.x.x`中 Prisma 的发布方案将遵循语义版本控制 ([SemVer](https://semver.org/)).

### Prisma 和语义版本控制 (SemVer)

#### SemVer 怎么控制?

语义版本 (SemVer) 使用以下版本升级规则 (引用 [SemVer](https://semver.org/) 说明):

_给定版本号 `MAJOR.MINOR.PATCH`, 递增:_

1. _`MAJOR` 版本：不兼容 API 变化时,_
1. _`MINOR` 版本：向后兼容方式添加功能时，_
1. _`PATCH` 版本：向后兼容的错误修复时_

#### 怎么使用 SemVer 对 Prisma 进行版本控制？

`3.x.x`版本开始, Prisma 严格遵守 [SemVer](https://semver.org/) 控制计划.

以下是 Prisma 如何使用 SemVer 控制的简要描述：

- 稳定表面的突破性变化 (即[正式发布](https://www.prisma.io/docs/about/releases#generally-available-ga)) 只有在新的 `MAJOR` 才会引入.
- 仍可在 `MINOR`推出破坏性更改，但是仅限于默认处于非活动状态的选择加入预览和早期功能 (例如， 通过预览功能标志或特定的选择选项或者新的 CLI 命令)。
- `MINOR`中发布的选择性加入的突破性更改, 例如预览版和早期版，只有正式版才有新的`MAJOR`发布 (不在要求选择加入) .

给定版本号 `MAJOR.MINOR.PATCH`, Prisma 版本数字递增如下:

1. `MAJOR` 版本：当重大产品更新**有突破性变化**正式发布时，`MAJOR`版本递增。
1. `MINOR` 版本：当发布后向兼容新功能的产品更新时，`MINOR`版本递增。有突破性变化的功能只有在**选择加入**的情况下才能引入，即早期和预览。
1. `PATCH` 版本：当功能 bug 修复后，`PATCH`版本递增。且始终**向后兼容**。

> **注意:** 在版本`2.28.0`之前，Prisma 没有严格遵循 SemVer 版本控制。这意味着在`2.MINOR.PATCH`范围内发布的`MINOR`版本可能包含突破性更改。了解有关 Prisma 采用 SemVer 更多信息, 查看[博客文章](https://www.prisma.io/blog/prisma-adopts-semver-strictly)。
