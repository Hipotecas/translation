## `User` 和 `Post` 数据模型

选择 `User` 和 `Post` 模型的原因如下：

- 这两种模型不需要特定领域的知识。
- 在 ORM 领域中，它们也经常被用作示例，使来自其他工具的用户对它们很熟悉。
- 拥有一致的模型会让读者更容易了解不同的概念，因为上下文切换会更少。
- 文档作者的决策和认知开销更少。使用相同的模型可以减少决策疲劳，并且在尝试解释概念时减少了一件需要担心的事情。

我们正在积极研究 [更多模式例子](https://github.com/prisma/docs/issues/1626) 为你提供常见据模型作为起点, 例如电子商务.
