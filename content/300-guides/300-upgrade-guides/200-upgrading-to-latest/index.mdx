<TopBlock>

为了升级到最新版本的 Prisma，你需要:

1. 在 GitHub 上查阅包含破坏性更新和新功能的的[版本发布说明](https://github.com/prisma/prisma/releases)。
2. **同时**升级下面这两个包的版本为统一版本:

   - [`prisma`](https://www.npmjs.com/package/prisma)
   - [`@prisma/client`](https://www.npmjs.com/package/@prisma/client)

3. 在合适的情况下升级。破坏性更新可能会要求你修改 Prisma schema 和部分使用 Prisma client 的代码。

<Tip>

[codemods](codemods) 会在有破坏性更新的情况下帮助你重构代码，比如 codemod 的 2.12.0 版本会自动将 `findOne` 重命名为 `findUnique`。

</Tip>

</TopBlock>

## 开启预览性功能

某些版本包含暂时没有生产版本的预览性功能，需要先开启才能使用。详细信息请查看下面的文档：

- [开启 Prisma Client preview features](../../../concepts/components/preview-features/client-preview-features)
- [开启 Prisma CLI preview features](../../../concepts/components/preview-features/cli-preview-features)

## <inlinecode>dev</inlinecode> 版本

标记为`dev`的 [Npm 版本](http://npm.github.io/publishing-pkgs-docs/updating/using-tags.html) 是该包的最新开发版本，就是最新的 github 主分支版本。

安装 `dev` 版本:

```terminal
npm install @prisma/client@dev prisma@dev
```

<Admonition type="alert">

不要在生产环境中使用 `dev` 版本 - 直到你想使用的功能或 bug 修复发布正式版本。例如某 bug 修复版本 `@prisma/client@2.23.0-dev.25` 会包括在 `@prisma/client@2.23.0`内。

</Admonition>
