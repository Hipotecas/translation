<TopBlock>

学习如何通过将Prisma连接到你的MongoDB数据库并生成一个Prisma客户端来访问数据库，从头开始创建一个新的Node.js或TypeScript项目。 以下教程向您介绍[Prisma CLI](../../../../concepts/components/prisma-cli)和[Prisma Client](../../../../concepts/components/prisma-client)。

</TopBlock>

## 前置条件

为了成功完成本指南，您需要:

- 在你的机器上安装[Node.js](https://nodejs.org/en/)
- 使用副本集部署访问MongoDB 4.2+服务器。我们推荐使用[MongoDB Atlas](https://www.mongodb.com/cloud/atlas)。

  <Admonition type="warning">

  MongoDB 数据库连接器使用事务来支持嵌套写入。事务**需要**一个[副本集](https://docs.mongodb.com/manual/tutorial/deploy-replica-set/)部署。部署副本集的最简单方法是使用[Atlas](https://docs.atlas.mongodb.com/getting-started/)。这是免费的开始。

  </Admonition>

确保已有数据库[连接URL](../../../../reference/database-reference/connection-urls)。如果您没有运行数据库服务器，只想探索Prisma，请查看[快速指南](../../../quickstart)。

> 具体的版本需求请参见[系统需求](../../../../reference/system-requirements)。

## 创建项目设置

作为第一步，创建一个项目并进入项目根目录:

```terminal copy
mkdir hello-prisma
cd hello-prisma
```

<SwitchTech technologies={['typescript', '*']}>

接下来，初始化一个 TypeScript 项目并将 Prisma CLI 添加到开发依赖：

```terminal copy
npm init -y
npm install prisma typescript ts-node @types/node --save-dev
```

这将为您的TypeScript应用程序创建一个带有初始设置的`package.json`。

接下来，创建一个`tsconfig.json`文件并在其中添加以下配置：

```json file=tsconfig.json copy
{
  "compilerOptions": {
    "sourceMap": true,
    "outDir": "dist",
    "strict": true,
    "lib": ["esnext"],
    "esModuleInterop": true
  }
}
```

</SwitchTech>

<SwitchTech technologies={['node', '*']}>

接下来，初始化一个Node.js项目，并添加Prisma CLI作为开发依赖项:

```terminal copy
npm init -y
npm install prisma --save-dev
```

这将创建一个带有Node.js应用程序初始设置的`package.json`。

</SwitchTech>

现在你可以通过前缀`npx`来调用Prisma CLI:

```terminal
npx prisma
```

接下来，通过使用以下命令创建[Prisma schema](../../../../concepts/components/prisma-schema)文件来设置Prisma项目：

```terminal copy
npx prisma init
```

此命令创建了一个名为的新目录`prisma`，其中包含一个名为`schema.prisma`的`.env`文件和项目根目录中的一个文件。schema.prisma包含带有数据库连接和 Prisma Client生成器的 Prisma 模式。 `.env`是一个[dotenv](https://github.com/motdotla/dotenv)文件，用于定义环境变量（用于数据库连接）。
