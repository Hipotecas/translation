<TopBlock>

了解如何通过将 Prisma 连接到数据库并生成用于数据库访问的 Prisma 客户端，将 Prisma 添加到现有 Node.js 或 TypeScript 项目。以下教程向您介绍了[Prisma CLI](../../../../concepts/components/prisma-cli)、[Prisma Client](../../../../concepts/components/prisma-client)和[Prisma Introspection](../../../../concepts/components/introspection)。

</TopBlock>

## 先决条件

为了成功完成本指南，您需要：

<SwitchTech technologies={['*', 'postgresql']}>

- [Node.js](https://nodejs.org/en/) 安装在你的机器上
- 一个正在运行的[PostgreSQL](https://www.postgresql.org/)数据库服务器和一个至少有一张表的数据库

</SwitchTech>

<SwitchTech technologies={['*', 'mysql']}>

- [Node.js](https://nodejs.org/en/) installed on your machine
- a [MySQL](https://www.mysql.com/) database server running

</SwitchTech>

> 具体的版本需求请参见[系统需求](../../../../reference/system-requirements)。

确保已有数据库[连接URL](../../../../reference/database-reference/connection-urls)。如果您没有运行数据库服务器，只想探索Prisma，请查看[快速入门](../../../quickstart)。

## 创建项目设置

第一步，创建项目并进入项目目录：

```terminal copy
mkdir hello-prisma
cd hello-prisma
```

<SwitchTech technologies={['typescript', '*']}>

接下来，初始化TypeScript项目并将Prisma CLI作为开发依赖项添加到该项目中：

```terminal copy
npm init -y
npm install prisma typescript ts-node @types/node --save-dev
```

这将为您的TypeScript应用程序创建一个带有初始设置的`package.json`。

接下来，创建一个`tsconfig.json`文件，并向其中添加以下配置：

```json file=tsconfig.json copy
{
  "compilerOptions": {
    "sourceMap": true,
    "outDir": "dist",
    "strict": true,
    "lib": ["esnext"],
    "esModuleInterop": true
  }
}
```

</SwitchTech>

<SwitchTech technologies={['node', '*']}>

接下来，初始化Node.js项目并将Prisma CLI作为开发依赖项添加到该项目中：

```terminal copy
npm init -y
npm install prisma --save-dev
```

这将创建一个带有Node.js应用程序初始设置的`package.json` 。

</SwitchTech>

现在你可以通过前缀`npx`来调用Prisma CLI:

```terminal
npx prisma
```

接下来，通过使用以下命令创建[Prisma schema](../../../../concepts/components/prisma-schema)文件来设置Prisma项目:

```terminal copy
npx prisma init
```

这个命令创建了一个名为`prisma`的新目录，其中包含一个名为`schema.prisma`的文件和一个位于项目根目录中的`.env`文件`schema.prisma`包含prisma模式以及数据库连接和prisma客户端生成器。
`.env`是一个[dotenv](https://github.com/motdotla/dotenv)用于定义环境变量的文件（用于数据库连接）。
