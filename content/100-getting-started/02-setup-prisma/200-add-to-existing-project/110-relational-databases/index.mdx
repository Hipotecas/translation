<TopBlock>

了解如何通过将 Prisma 连接到数据库并生成用于数据库访问的 Prisma 客户端，将 Prisma 添加到现有 Node.js 或 TypeScript 项目。以下教程向您介绍了[Prisma CLI](../../../../concepts/components/prisma-cli)、[Prisma Client](../../../../concepts/components/prisma-client)和[Prisma Introspection](../../../../concepts/components/introspection)。

</TopBlock>

## 先决条件

为了成功完成本指南，您需要：

<SwitchTech technologies={['*', 'postgresql']}>

- 一个现有的 Node.js 项目 `package.json`
- 安装[Node.js](https://nodejs.org/en/)在你的机器上
- 一个正在运行的[PostgreSQL](https://www.postgresql.org/)数据库服务器和一个至少有一张表的数据库

</SwitchTech>

<SwitchTech technologies={['*', 'mysql']}>

- 一个现有的 Node.js 项目 `package.json`
- 安装[Node.js](https://nodejs.org/en/)在你的机器上
- 一个正在运行的[MySQL](https://www.mysql.com/)数据库服务器和一个至少有一张表的数据库

</SwitchTech>

> 有关确切的版本要求，请参阅[系统要求](../../../../reference/system-requirements)。

确保您手头有您的数据库[连接 URL](../../../../reference/database-reference/connection-urls)（包括您的身份验证凭据）！如果您没有运行数据库服务器而只想探索 Prisma，请查看[快速入门](../../../quickstart)。

## 设置Prisma

第一步，导航到包含该`package.json`文件的项目目录。

接下来，将 Prisma CLI 作为开发依赖项添加到您的项目中：

```terminal copy
npm install prisma --save-dev
```

您现在可以通过添加前缀来调用 Prisma CLI `npx`：

```terminal
npx prisma
```

接下来，通过使用以下命令创建[Prisma schema](../../../../concepts/components/prisma-schema)文件模板来设置您的 Prisma 项目：

```terminal copy
npx prisma init
```

此命令创建了一个名为的新目录`prisma`，其中包含一个名为`schema.prisma`的`.env`文件和项目根目录中的一个文件。schema.prisma包含带有数据库连接和 Prisma 客户端生成器的 Prisma 模式。 `.env`是一个[dotenv](https://github.com/motdotla/dotenv)文件，用于定义环境变量（用于数据库连接）。
