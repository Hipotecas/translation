## 下一步

本节列出了您现在可以从这里采取的一些潜在的后续步骤。随意探索这些内容或阅读[简介](../../../../concepts/overview/what-is-prisma)页面以获取对 Prisma 的高级概述。

### 继续探索Prisma Client API

您可以使用 Prisma Client API 发送各种查询。查看[API 参考](../../../../concepts/components/prisma-client)并使用本指南中现有的数据库设置来试用它们。

<Tip>

您可以使用编辑器的自动完成功能来了解不同的API调用及其参数。自动完成通常是通过点击键盘上的<kbd>CTRL</kbd>+<kbd>SPACE</kbd>来调用。

</Tip>

<details><summary>展开以获取更多 Prisma 客户端 API 示例</summary>

对于您可以使用 Prisma Client 发送的更多查询，以下是一些建议：

**过滤所有`Post`包含的记录`"hello"`**

```js
const filteredPosts = await prisma.post.findMany({
  where: {
    OR: [
      { title: { contains: "hello" },
      { content: { contains: "hello" },
    ],
  },
})
```

**创建新`Post`记录并将其连接到现有`User`记录**

```js
const post = await prisma.post.create({
  data: {
    title: 'Join us for Prisma Day 2020',
    author: {
      connect: { email: 'alice@prisma.io' },
    },
  },
})
```

**使用 fluent 关系 API通过遍历关系检索`User`的`Post`记录**

```js
const posts = await prisma.profile
  .findUnique({
    where: { id: 1 },
  })
  .user()
  .posts()
```

**删除一条`User`记录**

```js
const deletedUser = await prisma.user.delete({
  where: { email: 'sarah@prisma.io' },
})
```

</details>

### 在 Prisma Studio 中探索数据

Prisma Studio 是数据库中数据的可视化编辑器。您可以通过两种方式运行它：

1. 在您的终端中运行`$ npx prisma studio`。
2. 从安装程序安装[桌面应用程序](https://github.com/prisma/studio/releases)。支持 Windows、macOS 和 Linux。

### 更改数据库架构（例如添加更多表）

要改进应用程序，您需要遵循本教程的相同流程：

1. 使用 SQL 手动调整数据库架构
2. 重新审视你的数据库
3. （可选）重新配置您的 Prisma 客户端 API
4. 重新生成 Prisma 客户端
