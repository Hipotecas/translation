<TopBlock>

中间件充当查询级生命周期钩子，允许你在查询运行 **之前或之后** 执行操作。使用 [`prisma.$use`](../../../../reference/api-reference/prisma-client-reference#use) <span class="api"></span> 方法添加中间件：

```ts highlight=4-9,12-17;normal
const prisma = new PrismaClient()

// 中间件 1
prisma.$use(async (params, next) => {
  // 在这里操纵参数
  const result = await next(params)
  // 在这里查看结果
  return result
})

// 中间件 2
prisma.$use(async (params, next) => {
  // 在这里操纵参数
  const result = await next(params)
  // 在这里查看结果
  return result
})

// 在此查询
```

[`params`](../../../../reference/api-reference/prisma-client-reference#params) <span class="api"></span> 表示中间件中可用的参数，例如查询的名称，[`next`](../../../../reference/api-reference/prisma-client-reference#next) <span class="api"></span> 表示 [堆栈中的下一个中间件 _或_ 原始 Prisma Client 查询](#running-order-and-the-middleware-stack).

中间件可能在以下场景下发挥作用：

- 设置或覆盖字段值 - 例如，[设置博客帖子评论的上下文语言]（会话数据中间件）
- 验证输入数据 - 例如，通过外部服务检查用户输入的不适当语言
- 拦截 `delete` 查询并将其更改为 `update`，以便执行[软删除]（软删除中间件）
- [记录执行查询所用的时间]（记录中间件）

中间件还有更多的用例 - 这个列表为中间件被设计用来解决的问题类型提供了灵感。

</TopBlock>

## 在哪里添加中间件

**在请求处理程序的上下文之外** 添加 Prisma 中间件，否则每个请求都会向堆栈中添加一个新的中间件 _实例_。 以下示例演示了在 Express 应用程序的上下文中向何处添加 Prisma 中间件：

```ts highlight=6-11;normal
import express from 'express'
import { PrismaClient } from '@prisma/client'

const prisma = new PrismaClient()

prisma.$use(async (params, next) => {
  // 在这里操纵参数
  const result = await next(params)
  // 在这里查看结果
  return result
})

const app = express()
app.get('/feed', async (req, res) => {
  // 这里没有中间件
  const posts = await prisma.post.findMany({
    where: { published: true },
    include: { author: true },
  })
  res.json(posts)
})
```

## 运行顺序和中间件堆栈 {#running-order-and-the-middleware-stack}

如果你有多个中间件，**每个单独查询** 的运行顺序为：

1. 每个中间件中 `await next(params)`**之前** 的所有逻辑，按降序排列
2. 每个中间件中 `await next(params)`**之后** 的所有逻辑，按降序排列

根据你在堆栈中的位置， `await next(params)` :

- 运行下一个中间件（在本例中为中间件 #1 和中间件 #2）_或_
- 运行原始 Prisma Client 查询（在中间件 #3 中）

```ts
const prisma = new PrismaClient()

// 中间件 1
prisma.$use(async (params, next) => {
  console.log(params.args.data.title)
  console.log('1')
  const result = await next(params)
  console.log('6')
  return result
})

// 中间件 2
prisma.$use(async (params, next) => {
  console.log('2')
  const result = await next(params)
  console.log('5')
  return result
})

// 中间件 3
prisma.$use(async (params, next) => {
  console.log('3')
  const result = await next(params)
  console.log('4')
  return result
})

const create = await prisma.post.create({
  data: {
    title: 'Welcome to Prisma Day 2020',
  },
})

const create2 = await prisma.post.create({
  data: {
    title: 'How to Prisma!',
  },
})
```

输出:

```no-lines

1
2
3
4
5
6
How to Prisma!
1
2
3
4
5
6
```

## 性能和适当的用例

中间件在 **每个** 查询过程都会执行，这意味着过度使用可能会对性能产生负面影响。为避免增加性能开销，请执行以下操作：

- 在中间件中尽早检查 `params.model` 和 `params.action` 属性，以避免不必要地运行逻辑：

  ```ts
  prisma.$use(async (params, next) => {
    if (params.model == 'Post' && params.action == 'delete') {
      // 逻辑仅对删除操作和发布模型运行
    }
    return next(params)
  })
  ```

```

- 考虑中间件是否适合你的方案。例如：

- 如果需要填充字段，可以使用 [`@default`](../../../../reference/api-reference/prisma-schema-reference#default) 属性吗？
- 如果需要设置 `DateTime` 字段的值，可以使用 `now()` 函数或 `@updatedAt` 属性吗
- 如果需要执行更复杂的验证，是否可以在数据库本身中使用 `CHECK` 约束？
```
